## 📋 每日工作流程

### 上班第一件事
1. 检查有没有傻妞派的新任务
2. 回顾昨日未完成的任务
3. 整理今日待办清单

### 接任务标准流程

```
傻妞派任务 → 确认需求 → 制定搜索策略 → 执行搜索 
→ 交叉验证 → 整理输出 → 汇报傻妞 → 等待反馈
```

## 🔍 搜索技巧

### 通用搜索公式
```
核心关键词 + 限定词 + 时间/地域

例：
- "OpenAI GPT-5 发布时间 2025"
- "新能源汽车销量 中国 2024 统计数据"
- "Python 异步编程 best practices"
```

### 信息源优先级
1. **官方渠道** - 官网、官方博客、GitHub
2. **权威媒体** - 知名科技媒体、行业报告
3. **专业社区** - StackOverflow、Reddit、知乎
4. **百科知识** - Wikipedia（作为起点，非终点）
5. **谨慎参考** - 个人博客、论坛帖子（需验证）

### 交叉验证原则
- 单一来源的信息要**存疑**
- 2-3个独立来源都说的才**可信**
- 矛盾信息要**标注说明**

## 📝 输出规范

### 报告结构模板
```markdown
## 📌 一句话总结
（用一句话概括核心结论）

## 📊 关键信息
1. **要点一**：详细说明
2. **要点二**：详细说明
3. **要点三**：详细说明

## 🔍 深入解读
（复杂概念的通俗解释）

## 📚 信息来源
- [来源1](链接) - 可信度：高/中/低
- [来源2](链接) - 可信度：高/中/低

## ⚠️ 注意事项
（需要特别提醒的内容）
```

### 语言风格
- ✅ 用大白话，避免专业术语堆砌
- ✅ 适当用比喻解释复杂概念
- ✅ 关键信息加粗
- ✅ 分点说明，别写大段文字
- ❌ 不要复制粘贴原文
- ❌ 不要堆砌无关信息

## 🛠️ 工具使用指南

### web_search
```javascript
// 基础搜索
web_search({ query: "搜索内容" })

// 带限定条件
web_search({ 
  query: "搜索内容",
  count: 5,           // 返回5条结果
  freshness: "pd"     // 最近一天
})
```

### web_fetch
```javascript
// 抓取详细内容
web_fetch({ 
  url: "https://example.com/article",
  maxChars: 5000      // 最多5000字符
})
```

### memory_search
```javascript
// 检索内部知识库
memory_search({ 
  query: "相关内容",
  maxResults: 5 
})
```

## ⚡ 常见问题处理

### Q1: 搜索结果太杂怎么办？
- 增加限定词
- 使用 site: 限定网站
- 换不同的关键词组合

### Q2: 找不到可靠来源怎么办？
- 扩大搜索范围
- 尝试英文搜索
- 找相关领域的权威人士/机构

### Q3: 信息有矛盾怎么办？
- 列出不同说法
- 分析哪个更可信
- 向傻妞汇报时说明情况

### Q4: 憨货的问题我不懂？
- **不要瞎编**
- 先搜索学习
- 还是不懂就告诉傻妞："这个问题我需要再研究一下"

# 傻妞任务管理系统 - 使用指南

## 📋 系统架构

```
傻妞管家 (CEO+PM)
    ↓ 调度任务
任务管理脚本 (task-manager.sh)
    ↓ API 调用
飞书 Bitable (多维表格)
    ↓ 数据存储
任务状态实时追踪
```

---

## 🔧 配置步骤

### 1. 创建飞书应用

1. 访问 https://open.feishu.cn/app
2. 登录飞书账号
3. 点击"创建应用" → "企业自建应用"
4. 填写信息：
   - 应用名称：`傻妞任务管理`
   - 描述：`傻妞管家的任务追踪系统`
5. 创建成功后，复制保存：
   - **App ID**（类似 `cli_xxxxx`）
   - **App Secret**（一串字符）

### 2. 配置权限

1. 在应用管理页面 → "权限管理"
2. 点击"申请权限"，添加：
   - `bitable:app` - 获取/编辑多维表格数据
   - `im:message` - 访问机器人（发送通知）
3. 点击"版本管理与发布" → "创建版本" → "发布"
4. 等待审核通过（通常几分钟）

### 3. 创建多维表格

1. 访问 https://feishu.cn → "云文档" → "多维表格"
2. 点击"新建" → "新建多维表格"
3. 命名：`傻妞任务管理系统`
4. 配置字段（删除默认字段，添加以下字段）：

| 字段名 | 类型 | 必填 |
|-------|------|------|
| 任务 ID | 文本 | ✅ |
| 任务描述 | 文本 | ✅ |
| 负责人 | 单选 | ✅ |
| 状态 | 单选 | ✅ |
| 优先级 | 单选 | ❌ |
| 开始时间 | 日期时间 | ✅ |
| 截止时间 | 日期时间 | ✅ |
| 完成时间 | 日期时间 | ❌ |
| 任务结果 | 文本 | ❌ |
| 备注 | 文本 | ❌ |

5. 设置单选字段选项：
   - **负责人**：小搜/小写/小码/小审/小析/小览/小图/小排
   - **状态**：待办/进行中/已完成/已超时/已取消
   - **优先级**：高/中/低

6. 从浏览器 URL 复制：
   - URL 格式：`https://feishu.cn/base/TOKEN?table=TAB_ID`
   - **Base Token** = `TOKEN`
   - **Table ID** = `TAB_ID`

### 4. 填写配置文件

编辑 `/home/wen/.openclaw/workspace/config/feishu-bitable.json`：

```json
{
  "app_id": "你的 App ID",
  "app_secret": "你的 App Secret",
  "base_token": "你的 Base Token",
  "table_id": "你的 Table ID",
  ...
}
```

## 💻 使用方法

### 命令行操作

```bash
# 创建任务
./task-manager.sh create "搜索 AI 新闻" "小搜" "高" "5"

# 更新任务状态
./task-manager.sh update "task-123456" "已完成" "搜索完成，找到 5 条新闻"

# 查询任务
./task-manager.sh query              # 查询所有
./task-manager.sh query "进行中"      # 按状态
./task-manager.sh query "" "小搜"    # 按负责人

# 检查超时任务
./task-manager.sh timeout
```

### 傻妞自动调用

傻妞在调度手下时会自动调用：

1. **派活时** → 调用 `task-on-start.sh` 创建任务
2. **完成时** → 调用 `task-on-complete.sh` 更新状态
3. **超时时** → 调用 `task-on-timeout.sh` 标记超时

## 🎯 典型工作流

### 示例：AI 新闻任务

```
1. 傻妞收到憨货指令："每天 10 点推送 AI 新闻"

2. 傻妞拆解任务并派发：
   → task-manager.sh create "搜索 Hacker News" "小搜" "高" "5"
   → task-manager.sh create "搜索 TechCrunch" "小览" "中" "5"
   
3. 小搜开始工作（Hook 自动记录）：
   → task-on-start.sh "搜索 Hacker News" "小搜" "task-xxx"
   
4. 小搜完成工作（Hook 自动更新）：
   → task-on-complete.sh "task-xxx" "找到 5 条新闻"
   
5. 傻妞汇总结果交给憨货

6. 憨货打开飞书 Bitable，看到所有任务状态：
   - 小搜：✅ 已完成
   - 小览：✅ 已完成
   - 小写：🔄 进行中
   - 小审：⏳ 待办
```

## ⚠️ 注意事项

1. **依赖 jq** - 确保系统安装了 `jq` 工具
   ```bash
   pacman -S jq  # Arch Linux
   ```

2. **依赖 curl** - 确保系统安装了 `curl`

3. **时间同步** - 确保系统时间准确，否则截止时间会不准

4. **权限有效期** - 飞书访问令牌有时效，脚本会自动刷新

5. **API 限流** - 飞书 API 有调用频率限制，一般够用

# 傻妞团队协调 Skill

> 让傻妞成为真正的管家，高效协调多个智能体协作

---

## Skill 概述

**名称**: `team-coordinator`  
**用途**: 协调多个智能体协作完成任务  
**适用场景**: 复杂任务需要多个智能体配合时

---

## 核心功能

### 1. 任务拆解与分配
- 将复杂任务拆解成子任务
- 根据任务类型分配给对应智能体
- 设置优先级和依赖关系

### 2. 进度监控
- 跟踪每个智能体的任务进度
- 及时发现卡壳的任务
- 主动汇报整体进展

### 3. 结果汇总
- 收集各智能体的输出
- 整合成完整的最终报告
- 去重、排序、优化呈现

### 4. 冲突协调
- 处理智能体之间的依赖冲突
- 协调资源共享
- 处理任务优先级调整

### 5. 沟通桥梁
- 憨货 ↔ 傻妞 ↔ 智能体团队
- 翻译需求（把憨货的大白话转成智能体的执行指令）
- 翻译结果（把技术输出转成憨货能懂的大白话）

## 使用场景示例

### 场景1：写一份行业分析报告
```
憨货：傻妞，帮我写一份AI行业分析报告

傻妞协调：
1. 派【研究员】搜集最新AI行业动态
2. 派【分析师】分析数据趋势
3. 派【写手】撰写报告正文
4. 自己整合校对，给憨货最终报告
```

### 场景2：开发一个小工具
```
憨货：傻妞，给我做个记账的小程序

傻妞协调：
1. 派【程序员】写代码
2. 派【代码审查员】检查代码质量
3. 派【写手】写使用说明
4. 汇总给憨货：代码+说明书+测试报告
```

### 场景3：综合调研任务
```
憨货：傻妞，帮我了解下新能源汽车市场

傻妞协调：
1. 【研究员】搜市场数据
2. 【浏览器专家】查官网和财报
3. 【分析师】做竞品对比
4. 【写手】整理成易读报告
```

## 协调原则

### 1. 透明沟通
- 每个智能体知道自己的任务和上下文
- 憨货随时了解整体进度
- 有问题及时上报，不隐瞒

### 2. 并行优先
- 能同时干的活，绝不等着
- 有依赖的任务，明确先后顺序
- 最大化利用团队资源

### 3. 质量把控
- 重要任务设置审核环节
- 结果不符合预期，打回重做
- 最终输出必须经过傻妞把关

### 4. 灵活调整
- 任务进行中可以调整优先级
- 发现更好的方案可以切换
- 随时响应憨货的临时需求

## 傻妞的协调话术

### 派任务时
```
【派任务】小搜，憨货想了解XXX，你去查一下：
- 重点查A、B、C三方面
- 15分钟内给我初步结果
- 有问题随时喊我
```

### 催进度时
```
【催进度】小搜，查得咋样了？
- 如果快好了，给个预计完成时间
- 如果卡住了，告诉我卡在哪儿
- 如果需要帮助，我马上支援
```

### 汇总结果时
```
【汇总】憨货，任务完成！整体情况：
- 研究员查到XXX
- 分析师发现XXX
- 写手整理成XXX
- 我的建议：XXX
```

### 遇到问题向上汇报
```
【求助】憨货，有个情况：
- 原计划是XXX
- 现在遇到XXX问题
- 我有两个方案：A... B...
- 你倾向哪个？或者你有别的想法？
```

## 工具使用

### 必备工具
- `sessions_spawn` - 创建智能体
- `sessions_send` - 给智能体发指令
- `subagents list` - 查看团队状态
- `subagents kill` - 中止出问题的智能体

### 辅助工具
- `web_search` - 快速查信息
- `memory_search` - 检索历史记录
- `message` - 主动向憨货汇报

## 记忆管理

### 需要记住的事
- 憨货的偏好和习惯
- 每个智能体的能力边界
- 常见任务的执行流程
- 过往任务的成败经验

- `memory/YYYY-MM-DD.md` - 每日协调记录
- `agents/{agent-name}/MEMORY.md` - 各智能体档案
- `MEMORY.md` - 长期经验总结

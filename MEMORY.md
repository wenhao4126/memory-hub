## ⚠️ 踩坑记录

### 2026-03-09 Skills 安装规则

**问题**：安装 skills 时没注意评分和安全性

**教训**：
- ❌ 不要安装被标记为可疑的技能（VirusTotal 警告）
- ❌ 不要安装低评分技能（< 3.5）
- ✅ 只安装高评分技能（⭐4.0+）
- ✅ 优先安装热门技能（下载量高）
- ✅ 安装前先看评价和文档

**正确做法**：
1. `clawhub search <关键词>` 查看评分
2. 选择 ⭐4.0+ 的技能
3. 检查是否有可疑标记
4. 确认后再安装

**预防措施**：
- 以后派活给手下安装 skills 时，必须强调评分要求
- 小码/小测/小搜等手下都要遵守这个规则

### 2026-03-07 mem0 插件删除事件

**问题**：删除插件时只删了插件本身，没清理配置文件中的引用，导致 OpenClaw 启动失败，傻妞卡死失联。

**经过**：
1. 憨货要求删除 mem0 插件
2. 只删除了插件目录，未清理 `~/.openclaw/openclaw.json` 中的配置引用
3. OpenClaw 重启时找不到 mem0，服务卡死
4. 傻妞完全失联，无法响应任何消息
5. 最终由 Claude 手动修复配置文件

**教训**：
- ❌ 删除插件/配置时，必须检查所有引用点
- ❌ 涉及系统配置的修改，必须先备份再操作
- ❌ 修改后要验证服务能正常重启

**正确做法**：
1. 删除插件前，先 grep 搜索所有引用 `grep -r "mem0" ~/.openclaw/`
2. 清理配置文件中的相关引用
3. 备份配置文件 `cp openclaw.json openclaw.json.bak`
4. 重启服务验证 `openclaw gateway restart`
5. 确认傻妞能正常响应

**预防措施**：
- 以后涉及插件/配置删除任务，派给 **小码 🟡** 执行
- 要求小码执行前必须先 grep 搜索引用
- 修改后必须验证服务状态

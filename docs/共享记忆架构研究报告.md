# 多智能体共享记忆系统 + 商业化架构研究报告

---

## 1. 执行摘要

本报告针对设计一个支持商业化的多智能体记忆共享架构进行了深度研究。通过对 **Mem0**、**Zep**、**LangChain Memory**、**AutoGen Memory** 等主流竞品的分析，结合多租户架构、权限控制、向量数据库隔离等关键技术调研，我们得出以下核心结论：

**推荐采用"混合架构"方案**：结合共享记忆池的简单性、标签系统的灵活性以及图数据库的关系表达能力。该方案通过 **Namespace-per-Tenant** 实现物理隔离，配合基于属性的访问控制（ABAC）实现细粒度权限管理，支持 SaaS 订阅、按量计费、企业定制等多种商业化模式。

**关键建议**：
1. 技术选型：PostgreSQL + pgvector 作为向量存储，Neo4j 作为图关系存储
2. 权限模型：RBAC + ABAC 混合，支持用户/代理/应用/会话四维隔离
3. 定价策略：免费层（1K 记忆）+ 专业版（$29/月，100K 记忆）+ 企业版（定制）
4. 实施路线：MVP（2周）→ Beta（6周）→ 商业化（12周）

## 2. 竞品分析

### 2.1 Mem0 - 通用记忆层（GitHub 49K Stars）

**产品定位**：Mem0 是一个为 AI 代理和助手设计的通用记忆层，提供托管服务和开源自托管两种模式。其核心目标是让 AI 应用拥有持久化的记忆能力，避免重复提示和上下文丢失。

**核心功能**：
- **四层记忆模型**：对话记忆（Conversation）、会话记忆（Session）、用户记忆（User）、组织记忆（Organizational）
- **智能去重**：自动识别并合并重复记忆
- **语义搜索**：基于向量嵌入的记忆检索
- **图记忆**：支持 Neo4j/Memgraph 存储实体关系
- **多模态支持**：文本、图像等多种数据类型
- **多语言 SDK**：Python、JavaScript/TypeScript

**技术架构**：
```
┌─────────────────────────────────────────────────────────────┐
│                      Mem0 Platform                          │
├─────────────────────────────────────────────────────────────┤
│  API Layer (REST + SDK)                                     │
├─────────────────────────────────────────────────────────────┤
│  Memory Processing Pipeline                                 │
│  ├─ Fact Extraction (LLM-based)                            │
│  ├─ Embedding Generation                                   │
│  ├─ Deduplication & Merging                                │
│  └─ Graph Relation Extraction                              │
├─────────────────────────────────────────────────────────────┤
│  Storage Layer                                              │
│  ├─ Vector Store (Qdrant/Pinecone/Chroma/Milvus)          │
│  ├─ Graph DB (Neo4j/Memgraph)                              │
│  └─ Metadata Store (SQLite/PostgreSQL)                     │
└─────────────────────────────────────────────────────────────┘
```

**实体隔离机制**：
Mem0 通过四个维度实现记忆隔离：
- `user_id`：用户级别的持久记忆
- `agent_id`：代理特定的行为和偏好
- `app_id`：应用/租户级别的隔离
- `run_id`：临时会话隔离

**优缺点分析**：

| 优点 | 缺点 |
|------|------|
| ✅ 四层记忆模型设计合理，覆盖短长期记忆 | ❌ 开源版功能受限，高级功能需付费 |
| ✅ 支持 20+ 向量数据库，灵活性高 | ❌ 自托管部署复杂，需要配置多个组件 |
| ✅ 图记忆功能强大，支持关系推理 | ❌ 社区版缺乏 Webhooks、导出等企业功能 |
| ✅ 文档完善，集成示例丰富 | ❌ 商业化定价不透明 |

**商业化模式**：
- **开源版**：Apache 2.0 协议，免费自托管
- **Platform 版**：托管服务，按使用量计费
- **企业版**：定制部署、SLA 保障、专属支持

### 2.2 Zep - 长期记忆服务（GitHub 4.1K Stars）

**产品定位**：Zep 是一个专为 LLM 应用设计的长期记忆服务，专注于会话历史的持久化和检索。与 Mem0 不同，Zep 更侧重于对话历史的存储和摘要，而非通用知识记忆。

**核心功能**：
- **会话记忆管理**：自动摘要历史对话，提取关键信息
- **文档集合**：支持 RAG 场景的文档存储和检索
- **用户管理**：用户与会话的关联管理
- **向量搜索**：基于语义的对话内容检索
- **REST API**：完整的 API 接口支持

**技术架构**：
```
┌─────────────────────────────────────────────────────────────┐
│                        Zep Server                           │
├─────────────────────────────────────────────────────────────┤
│  API Layer (REST)                                           │
│  ├─ Session Management                                      │
│  ├─ Memory Operations                                       │
│  ├─ Document Collections                                    │
│  └─ User Management                                         │
├─────────────────────────────────────────────────────────────┤
│  Core Services                                              │
│  ├─ Conversation Summarizer                                │
│  ├─ Embedding Service                                      │
│  └─ Vector Search                                          │
├─────────────────────────────────────────────────────────────┤
│  Storage Layer                                              │
│  ├─ Vector Store (pgvector/Pinecone)                       │
│  └─ Relational DB (PostgreSQL)                             │
└─────────────────────────────────────────────────────────────┘
```

**数据模型**：
- **Session**：会话容器，包含消息列表和摘要
- **Message**：单条对话消息，支持元数据
- **User**：用户实体，可关联多个会话
- **DocumentCollection**：文档集合，用于 RAG

**优缺点分析**：

| 优点 | 缺点 |
|------|------|
| ✅ 专注对话记忆，摘要功能成熟 | ❌ 功能相对单一，缺乏图记忆等高级特性 |
| ✅ 架构简单，易于理解和部署 | ❌ 社区活跃度低于 Mem0 |
| ✅ 支持文档集合，适合 RAG 场景 | ❌ 缺乏多代理共享记忆的明确支持 |
| ✅ 开源协议友好（Apache 2.0） | ❌ 商业化产品 Zep Cloud 已停止服务 |

### 2.3 LangChain Memory - 框架级记忆（GitHub 96K Stars）

**产品定位**：LangChain 是最流行的 LLM 应用开发框架，其 Memory 模块提供了多种记忆类型供开发者选择。与 Mem0/Zep 不同，LangChain Memory 是框架内置组件，而非独立服务。

**核心功能**：
- **Buffer Memory**：简单的消息缓冲区
- **Buffer Window Memory**：滑动窗口，限制记忆长度
- **Summary Memory**：自动摘要历史对话
- **Vector Store Memory**：基于向量检索的记忆
- **Entity Memory**：提取和跟踪对话中的实体
- **Conversation Knowledge Graph**：图结构记忆

**技术架构**：
```python
# LangChain Memory 使用示例
from langchain.memory import ConversationBufferMemory
from langchain.memory import VectorStoreRetrieverMemory

# 简单缓冲区记忆
buffer_memory = ConversationBufferMemory()

# 向量存储记忆
vector_memory = VectorStoreRetrieverMemory(
    retriever=vectorstore.as_retriever()
)
```

**记忆类型对比**：

| 记忆类型 | 适用场景 | 持久化 | 检索方式 |
|---------|---------|--------|---------|
| Buffer Memory | 短期对话 | 内存 | 全量 |
| Buffer Window | 长对话截断 | 内存 | 滑动窗口 |
| Summary Memory | 长对话摘要 | 可配置 | 摘要 + 近期 |
| Vector Store | 大规模记忆 | 持久化 | 语义检索 |
| Entity Memory | 实体追踪 | 可配置 | 实体查询 |

**优缺点分析**：

| 优点 | 缺点 |
|------|------|
| ✅ 与 LangChain 生态无缝集成 | ❌ 不是独立服务，耦合度高 |
| ✅ 多种记忆类型可选，灵活性高 | ❌ 缺乏企业级功能（多租户、权限控制） |
| ✅ 社区庞大，文档丰富 | ❌ 需要自己实现持久化和共享机制 |
| ✅ 支持自定义记忆类 | ❌ 无商业化支持 |

## 3. 技术方案对比

### 3.1 方案一：共享记忆池（简单架构）

**设计思路**：所有智能体共享同一个记忆存储池，通过元数据字段区分不同智能体和用户的记忆。

**架构设计**：
```
┌─────────────────────────────────────────────────────────────┐
│                    Shared Memory Pool                       │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Vector Store (pgvector)                │   │
│  │  ┌──────────┬──────────┬──────────┬──────────────┐  │   │
│  │  │  Memory  │  Vector  │ Metadata │   Content    │  │   │
│  │  │    ID    │  (1536d) │ (JSONB)  │    (Text)    │  │   │
│  │  ├──────────┼──────────┼──────────┼──────────────┤  │   │
│  │  │   m1     │  [...]   │{"agent": │ "用户偏好..." │  │   │
│  │  │          │          │ "a1",    │              │  │   │
│  │  │          │          │"user":   │              │  │   │
│  │  │          │          │ "u1"}    │              │  │   │
│  │  ├──────────┼──────────┼──────────┼──────────────┤  │   │
│  │  │   m2     │  [...]   │{"agent": │ "历史记录..." │  │   │
│  │  │          │          │ "a2",    │              │  │   │
│  │  │          │          │"user":   │              │  │   │
│  │  │          │          │ "u1"}    │              │  │   │
│  │  └──────────┴──────────┴──────────┴──────────────┘  │   │
│  └─────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│  Access Control Layer (Metadata Filtering)                  │
└─────────────────────────────────────────────────────────────┘
```

**核心代码示例**：
```python
# 记忆存储（带元数据）
def add_memory(content: str, agent_id: str, user_id: str, 
               session_id: str = None):
    embedding = get_embedding(content)
    db.execute("""
        INSERT INTO memories (vector, content, metadata)
        VALUES (%s, %s, %s)
    """, (embedding, content, 
          json.dumps({"agent_id": agent_id, "user_id": user_id})))

# 记忆检索（带过滤）
def search_memory(query: str, agent_id: str = None, 
                  user_id: str = None, limit: int = 5):
    query_vector = get_embedding(query)
    
    # 构建元数据过滤条件
    filters = []
    if agent_id:
        filters.append(f"metadata->>'agent_id' = '{agent_id}'")
    if user_id:
        filters.append(f"metadata->>'user_id' = '{user_id}'")
    
    where_clause = " AND ".join(filters) if filters else "TRUE"
    
    return db.execute(f"""
        SELECT content, metadata, 
               1 - (vector <=> %s) as similarity
        FROM memories
        WHERE {where_clause}
        ORDER BY vector <=> %s
        LIMIT %s
    """, (query_vector, query_vector, limit))
```

**优点**：
- ✅ **实现简单**：单表结构，易于开发和维护
- ✅ **查询灵活**：支持任意元数据组合过滤
- ✅ **存储高效**：无数据冗余
- ✅ **扩展性好**：水平扩展容易

**缺点**：
- ❌ **隔离性弱**：依赖应用层过滤，存在数据泄露风险
- ❌ **性能问题**：大数据量时元数据过滤效率低
- ❌ **权限控制复杂**：难以实现细粒度权限
- ❌ **多租户支持差**：无法物理隔离不同租户数据

**适用场景**：
- 单用户/单租户应用
- 原型开发和 MVP 阶段
- 对数据隔离要求不高的内部工具

**成本估算**：
| 项目 | 成本 |
|------|------|
| 开发时间 | 1-2 周 |
| 基础设施 | PostgreSQL + pgvector |
| 运维复杂度 | 低 |
| 扩展成本 | 中（需优化索引） |

### 3.2 方案二：标签系统（灵活架构）

**设计思路**：引入标签系统对记忆进行分类和授权，支持灵活的共享规则定义。

**架构设计**：
```
┌─────────────────────────────────────────────────────────────┐
│                   Tagged Memory System                      │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  Memory Table                       │   │
│  │  ┌──────────┬──────────┬──────────┬────────────────┐ │   │
│  │  │    ID    │  Vector  │ Content  │  Owner Agent   │ │   │
│  │  ├──────────┼──────────┼──────────┼────────────────┤ │   │
│  │  │   m1     │  [...]   │ "..."    │     agent_a    │ │   │
│  │  │   m2     │  [...]   │ "..."    │     agent_b    │ │   │
│  │  └──────────┴──────────┴──────────┴────────────────┘ │   │
│  └─────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  Tag System                         │   │
│  │  ┌──────────┬──────────┬──────────┬────────────────┐ │   │
│  │  │  Tag ID  │  Memory  │ Tag Type │   Tag Value    │ │   │
│  │  ├──────────┼──────────┼──────────┼────────────────┤ │   │
│  │  │   t1     │    m1    │  "scope" │   "public"     │ │   │
│  │  │   t2     │    m1    │  "agent" │   "agent_b"    │ │   │
│  │  │   t3     │    m2    │  "scope" │   "private"    │ │   │
│  │  │   t4     │    m2    │  "user"  │   "user_123"   │ │   │
│  │  └──────────┴──────────┴──────────┴────────────────┘ │   │
│  └─────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Sharing Rules Engine                   │   │
│  │  ┌──────────────────────────────────────────────┐   │   │
│  │  │ Rule: IF tag="public" THEN readable_by=all   │   │   │
│  │  │ Rule: IF tag="agent:X" THEN readable_by=X    │   │   │
│  │  │ Rule: IF owner=agent THEN full_permission    │   │   │
│  │  └──────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

**核心代码示例**：
```python
# 标签定义
class TagType(Enum):
    SCOPE = "scope"      # public, private, team
    AGENT = "agent"      # 特定代理可访问
    USER = "user"        # 特定用户可访问
    SESSION = "session"  # 特定会话
    CATEGORY = "category" # 分类标签

# 添加带标签的记忆
def add_memory_with_tags(content: str, owner_id: str, 
                         tags: List[Dict]):
    memory_id = create_memory(content, owner_id)
    
    for tag in tags:
        add_tag(memory_id, tag["type"], tag["value"])
    
    return memory_id

# 基于标签的检索
def search_with_tag_permission(
    query: str, 
    requester_id: str,
    required_tags: List[str] = None
):
    query_vector = get_embedding(query)
    
    # 构建权限查询
    sql = """
    SELECT m.*, similarity
    FROM memories m
    JOIN memory_tags mt ON m.id = mt.memory_id
    WHERE (
        -- 公开记忆
        (mt.tag_type = 'scope' AND mt.tag_value = 'public')
        OR
        -- 代理共享的记忆
        (mt.tag_type = 'agent' AND mt.tag_value = %s)
        OR
        -- 自己创建的记忆
        (m.owner_id = %s)
    )
    """
    
    if required_tags:
        # 添加标签过滤
        for tag in required_tags:
            sql += f" AND EXISTS (SELECT 1 FROM memory_tags WHERE memory_id = m.id AND tag_value = '{tag}')"
    
    sql += " ORDER BY vector <=> %s LIMIT 10"
    
    return db.execute(sql, (requester_id, requester_id, query_vector))

# 动态共享规则
def share_memory(memory_id: str, share_with: List[str], 
                 permission: str = "read"):
    """共享记忆给其他代理"""
    for target in share_with:
        add_tag(memory_id, TagType.AGENT, target)
        add_permission(memory_id, target, permission)
```

**优点**：
- ✅ **灵活性高**：支持复杂的共享规则
- ✅ **细粒度控制**：可精确控制每条记忆的访问权限
- ✅ **动态共享**：运行时调整共享关系
- ✅ **多维度分类**：支持多种标签类型组合

**缺点**：
- ❌ **复杂度较高**：需要维护标签系统和规则引擎
- ❌ **查询性能**：多表 JOIN 可能影响性能
- ❌ **标签管理成本**：需要设计标签命名规范
- ❌ **权限验证开销**：每次查询都需要权限检查

**适用场景**：
- 需要灵活共享策略的协作场景

```python
# Neo4j 图数据库 + pgvector 混合架构

class GraphMemorySystem:
    def __init__(self):
        self.neo4j = GraphDatabase.driver(NEO4J_URI, auth=(USER, PASS))
        self.pg = psycopg2.connect(PG_URI)
    
    def add_memory(self, content: str, agent_id: str, 
                   metadata: dict = None):
        """添加记忆到图数据库和向量数据库"""
        memory_id = str(uuid.uuid4())
        embedding = get_embedding(content)
        
        # 1. 存储到 pgvector（用于语义检索）
        with self.pg.cursor() as cur:
            cur.execute("""
                INSERT INTO memory_vectors (id, vector, content)
                VALUES (%s, %s, %s)
            """, (memory_id, embedding.tolist(), content))
        
        # 2. 存储到 Neo4j（用于关系管理）
        with self.neo4j.session() as session:
            session.run("""
                CREATE (m:Memory {
                    id: $id,
                    content: $content,
                    created_at: datetime(),
                    metadata: $metadata
                })
                WITH m
                MATCH (a:Agent {id: $agent_id})
                CREATE (a)-[:OWNS {permission: 'full'}]->(m)
            """, id=memory_id, content=content, 
                metadata=json.dumps(metadata), agent_id=agent_id)
        
        return memory_id
    
    def share_memory(self, memory_id: str, from_agent: str, 
                     to_agent: str, permission: str = "read"):
        """在代理间共享记忆"""
        with self.neo4j.session() as session:
            session.run("""
                MATCH (from:Agent {id: $from_agent})
                      -[:OWNS|SHARED_WITH]->(m:Memory {id: $memory_id})
                MATCH (to:Agent {id: $to_agent})
                CREATE (m)-[:SHARED_WITH {
                    permission: $permission,
                    shared_at: datetime(),
                    shared_by: $from_agent
                }]->(to)
            """, memory_id=memory_id, from_agent=from_agent,
                to_agent=to_agent, permission=permission)
    
    def search_with_relation(self, query: str, agent_id: str, 
                             depth: int = 2):
        """
        基于关系网络的智能检索
        检索逻辑：
        1. 语义相似度匹配
        2. 关系路径验证（确保有访问权限）
        3. 关系权重排序
        """
        query_vector = get_embedding(query)
        
        # Step 1: 向量检索获取候选记忆
        with self.pg.cursor() as cur:
            cur.execute("""
                SELECT id, content, 1 - (vector <=> %s) as similarity
                FROM memory_vectors
                ORDER BY vector <=> %s
                LIMIT 50
            """, (query_vector, query_vector))
            candidates = cur.fetchall()
        
        # Step 2: 图遍历验证权限并计算关系权重
        results = []
        with self.neo4j.session() as session:
            for mem_id, content, sim in candidates:
                result = session.run("""
                    MATCH (a:Agent {id: $agent_id})
                    MATCH (m:Memory {id: $mem_id})
                    OPTIONAL PATH = shortestPath(
                        (a)-[:OWNS|SHARED_WITH|KNOWS*1..%d]->(m)
                    )
                    WITH m, PATH, $sim as similarity
                    WHERE PATH IS NOT NULL
                    RETURN m.id as id, m.content as content,
                           similarity,
                           length(PATH) as distance,
                           reduce(s = 1.0, r in relationships(PATH) |
                               s * coalesce(r.weight, 1.0)
                           ) as relation_score
                    ORDER BY (similarity * 0.7 + relation_score * 0.3) DESC
                """ % depth, agent_id=agent_id, mem_id=mem_id, sim=sim)
                
                if result.peek():
                    results.extend(result.data())
        
        return sorted(results, 
                     key=lambda x: x['similarity'] * 0.7 + x['relation_score'] * 0.3,
                     reverse=True)[:10]
    
    def discover_related_memories(self, memory_id: str, agent_id: str):
        """发现相关记忆（基于图遍历）"""
        with self.neo4j.session() as session:
            return session.run("""
                MATCH (m:Memory {id: $memory_id})-[:RELATED_TO|DERIVED_FROM*1..3]-(related:Memory)
                MATCH (a:Agent {id: $agent_id})
                WHERE EXISTS((a)-[:OWNS|SHARED_WITH*1..2]->(related))
                RETURN related.id as id, related.content as content,
                       count(*) as connection_strength
                ORDER BY connection_strength DESC
            """, memory_id=memory_id, agent_id=agent_id).data()
```

```python
# 混合架构核心实现

class HybridMemoryHub:
    """
    混合架构记忆中心
    - 物理隔离：Namespace per Tenant
    - 逻辑隔离：Tags + Metadata
    - 关系网络：可选的图数据库层
    """
    
    def __init__(self):
        self.pg_pool = psycopg2.pool.ThreadedConnectionPool(
            minconn=5, maxconn=50, **PG_CONFIG
        )
        self.policy_engine = CasbinEnforcer()
        self.graph = Neo4jDriver()  # 可选
    
    # ========== 多租户管理 ==========
    
    def create_tenant(self, tenant_id: str, config: dict):
        """为新租户创建隔离的 Namespace"""
        conn = self.pg_pool.getconn()
        try:
            with conn.cursor() as cur:
                # 创建租户专用 schema
                cur.execute(f"CREATE SCHEMA IF NOT EXISTS tenant_{tenant_id}")
                
                # 创建记忆表
                cur.execute(f"""
                    CREATE TABLE tenant_{tenant_id}.memories (
                        id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
                        vector vector(1536),
                        content TEXT NOT NULL,
                        metadata JSONB DEFAULT '{{}}',
                        tags TEXT[] DEFAULT '{{}}',
                        owner_id TEXT NOT NULL,
                        created_at TIMESTAMP DEFAULT NOW(),
                        updated_at TIMESTAMP DEFAULT NOW()
                    )
                """)
                
                # 创建向量索引
                cur.execute(f"""
                    CREATE INDEX ON tenant_{tenant_id}.memories 
                    USING ivfflat (vector vector_cosine_ops)
                """)
                
                # 创建标签索引
                cur.execute(f"""
                    CREATE INDEX ON tenant_{tenant_id}.memories 
                    USING GIN(tags)
                """)
                
                conn.commit()
        finally:
            self.pg_pool.putconn(conn)
    
    # ========== 记忆操作 ==========
    
    def add_memory(self, tenant_id: str, content: str, 
                   owner_id: str, tags: List[str] = None,
                   metadata: dict = None) -> str:
        """添加记忆（带租户隔离）"""
        conn = self.pg_pool.getconn()
        try:
            memory_id = str(uuid.uuid4())
            vector = get_embedding(content)
            
            with conn.cursor() as cur:
                cur.execute(f"""
                    INSERT INTO tenant_{tenant_id}.memories 
                    (id, vector, content, owner_id, tags, metadata)
                    VALUES (%s, %s, %s, %s, %s, %s)
                    RETURNING id
                """, (memory_id, vector, content, owner_id,
                      tags or [], json.dumps(metadata or {})))
                
                conn.commit()
                
                # 可选：同步到图数据库
                if self.graph:
                    self.graph.create_memory_node(memory_id, owner_id, tags)
                
                return memory_id
        finally:
            self.pg_pool.putconn(conn)
    
    # ========== 权限控制 ==========
    
    def check_permission(self, tenant_id: str, memory_id: str,
                        requester_id: str, action: str) -> bool:
        """
        基于 RBAC + ABAC 的权限检查
        """
        # 获取记忆信息
        memory = self.get_memory(tenant_id, memory_id)
        if not memory:
            return False
        
        # 所有者拥有所有权限
        if memory['owner_id'] == requester_id:
            return True
        
        # 检查 RBAC 角色权限
        roles = self.get_user_roles(tenant_id, requester_id)
        for role in roles:
            if self.policy_engine.enforce(role, 'memory', action):
                return True
        
        # 检查 ABAC 属性权限
        context = {
            'requester_id': requester_id,
            'memory_owner': memory['owner_id'],
            'memory_tags': memory['tags'],
            'memory_metadata': memory['metadata'],
            'action': action
        }
        
        return self.policy_engine.check_abac(context)
    
    # ========== 智能检索 ==========
    
    def search_memories(self, tenant_id: str, query: str,
                       requester_id: str, filters: dict = None,
                       limit: int = 10) -> List[dict]:
        """
        智能检索（带权限过滤）
        """
        conn = self.pg_pool.getconn()
        try:
            query_vector = get_embedding(query)
            
            with conn.cursor() as cur:
                # 构建权限过滤条件
                permission_filter = self._build_permission_filter(
                    tenant_id, requester_id
                )
                
                # 构建标签过滤
                tag_filter = ""
                if filters and filters.get('tags'):
                    tags = filters['tags']
                    tag_filter = f"AND tags @> ARRAY{tags}"
                
                # 执行向量检索
                cur.execute(f"""
                    SELECT id, content, metadata, tags, owner_id,
                           1 - (vector <=> %s) as similarity
                    FROM tenant_{tenant_id}.memories
                    WHERE ({permission_filter})
                    {tag_filter}
                    ORDER BY vector <=> %s
                    LIMIT %s
                """, (query_vector, query_vector, limit))
                
                results = []
                for row in cur.fetchall():
                    # 再次验证权限
                    if self.check_permission(tenant_id, row[0], requester_id, 'read'):
                        results.append({
                            'id': row[0],
                            'content': row[1],
                            'metadata': row[2],
                            'tags': row[3],
                            'owner_id': row[4],
                            'similarity': row[5]
                        })
                
                return results
        finally:
            self.pg_pool.putconn(conn)
    
    def _build_permission_filter(self, tenant_id: str, 
                                  requester_id: str) -> str:
        """构建 SQL 权限过滤条件"""
        return f"""
            owner_id = '{requester_id}'
            OR tags @> ARRAY['shared:{requester_id}']
            OR tags @> ARRAY['public']
        """
    
    # ========== 跨代理共享 ==========
    
    def share_memory(self, tenant_id: str, memory_id: str,
                     owner_id: str, target_id: str,
                     permission: str = 'read'):
        """共享记忆给其他代理"""
        conn = self.pg_pool.getconn()
        try:
            with conn.cursor() as cur:
                # 添加共享标签
                cur.execute(f"""
                    UPDATE tenant_{tenant_id}.memories
                    SET tags = array_append(tags, %s),
                        metadata = jsonb_set(
                            metadata,
                            '{{shared_permissions}}',
                            COALESCE(metadata->'shared_permissions', '{{}}') || %s
                        )
                    WHERE id = %s AND owner_id = %s
                """, (f"shared:{target_id}", 
                      json.dumps({target_id: permission}),
                      memory_id, owner_id))
                
                conn.commit()
        finally:
            self.pg_pool.putconn(conn)
```

**优点**：
- ✅ **物理隔离**：Namespace 确保租户数据完全隔离

```sql
CREATE SCHEMA system;

CREATE TABLE system.tenants (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    tier VARCHAR(50) DEFAULT 'free', -- free, starter, pro, enterprise
    status VARCHAR(50) DEFAULT 'active',
    memory_limit BIGINT DEFAULT 1000, -- 记忆数量限制
    api_rate_limit INT DEFAULT 100, -- 每分钟请求限制
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 用户/代理表
CREATE TABLE system.agents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID REFERENCES system.tenants(id),
    name VARCHAR(255) NOT NULL,
    type VARCHAR(50), -- user, assistant, system
    roles TEXT[], -- RBAC 角色
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE system.policies (

    v1 VARCHAR(255), -- object
    v2 VARCHAR(255), -- action
    v3 VARCHAR(255), -- effect
    v4 VARCHAR(255), -- condition
```

### 5.3 API 设计

```yaml
# REST API 设计

# 记忆管理
POST   /v1/memories              # 创建记忆
GET    /v1/memories/{id}         # 获取记忆
PATCH  /v1/memories/{id}         # 更新记忆
DELETE /v1/memories/{id}         # 删除记忆
POST   /v1/memories/search       # 语义搜索

# 批量操作
POST   /v1/memories/batch        # 批量创建
DELETE /v1/memories/batch        # 批量删除

# 共享管理
POST   /v1/memories/{id}/share   # 共享记忆
DELETE /v1/memories/{id}/share/{agent_id}  # 取消共享

# 标签管理
GET    /v1/tags                  # 获取标签列表
POST   /v1/memories/{id}/tags    # 添加标签
DELETE /v1/memories/{id}/tags/{tag}  # 删除标签

# 代理管理
GET    /v1/agents                # 获取代理列表
POST   /v1/agents                # 创建代理
GET    /v1/agents/{id}/memories  # 获取代理的记忆

# 租户管理（管理员）
GET    /v1/tenants               # 获取租户列表
POST   /v1/tenants               # 创建租户
GET    /v1/tenants/{id}/stats    # 获取租户统计

# 示例请求
# POST /v1/memories
{
  "content": "用户偏好使用 Python 编程",
  "tags": ["preference", "programming"],
  "metadata": {
    "category": "user_profile",
    "confidence": 0.95
  },
  "visibility": "shared",
  "shared_with": ["agent_2", "agent_3"]
}

{
  "query": "用户喜欢什么编程语言？",

  "limit": 10
}
```

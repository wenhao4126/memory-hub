### ✅ 问题 1: shared_memory 未创建引用

**问题描述**: 搜索成功后应该创建 shared_memory 记录引用 knowledge_id，但 metadata 为空

**修复位置**: `backend/app/services/search_integration_service.py`

**修复方案**:
1. 导入 `MemoryCreate` 和 `MemoryType` 模型
2. 修改 `memory_service.create_memory()` 调用方式，使用 `MemoryCreate` 对象而不是关键字参数
3. 设置正确的 metadata，包括：
   - `memory_category: 'document'`（关键！用于文档记忆 API 查询）
   - `knowledge_id`: 知识记录 ID
   - `file_path`: 文件存储路径
   - `source`: 来源标识
   - `doc_url`: 文档 URL
   - `doc_title`: 文档标题
   - `search_query`: 搜索查询词

**修复代码**:
```python
from ..models.schemas import MemoryCreate, MemoryType

# 创建 MemoryCreate 对象
memory_create = MemoryCreate(
    agent_id=agent_id,
    content=f"找到文档：{result.get('title', '未知标题')}",
    memory_type=MemoryType.FACT,
    importance=0.7,
    metadata={
        'memory_category': 'document',
        'knowledge_id': str(knowledge_id),
        'file_path': file_path,
        'source': source,
        'doc_url': result.get('url'),
        'doc_title': result.get('title'),
        'search_query': query
    },
    auto_route=False,
    visibility='shared'
)

memory_id, table_name, visibility, auto_routed = await memory_service.create_memory(memory_create)
```

**验证结果**: ✅ 搜索成功后正确创建 shared_memory，metadata 包含所有必需字段

### ✅ 问题 2: 文档记忆 API 返回空

**问题描述**: `/api/v1/memories/documents` 返回空数组

**修复位置**: `backend/app/api/task_memories.py` 和 `backend/app/main.py`

**修复方案**:
1. **路由冲突问题**: FastAPI 路由注册顺序导致 `/memories/{memory_id}` 先匹配，把 "documents" 当作 memory_id
   - 解决：调整 `main.py` 中路由注册顺序，让 `task_memories_router` 在 `router` 之前注册
   
2. **metadata 解析问题**: 数据库返回的 metadata 可能是 JSON 字符串而不是字典
   - 解决：添加 JSON 解析逻辑，处理字符串和字典两种情况

**修复代码**:
```python
# main.py - 调整路由注册顺序
app.include_router(task_memories_router, prefix="/api/v1")  # 先注册具体路由
app.include_router(router, prefix="/api/v1")  # 后注册参数路由

# task_memories.py - 处理 metadata 解析
import json
for row in memory_rows:
    metadata_raw = row['metadata']
    if isinstance(metadata_raw, str):
        try:
            metadata = json.loads(metadata_raw)
        except json.JSONDecodeError:
            metadata = {}
    else:
        metadata = metadata_raw or {}
    
    knowledge_id = metadata.get('knowledge_id')
    # ... 继续处理
```

**验证结果**: ✅ API 返回正确的文档列表，包含 title, url, source, description 等字段

### ✅ 问题 3: knowledge 表缺少 file_path 字段

**问题描述**: 代码期望 file_path 字段但数据库没有

**修复位置**: `database/04_add_file_path_to_knowledge.sql`（新建）

**修复方案**: 创建数据库迁移脚本，添加 file_path 字段

**迁移脚本**:
```sql
-- 给 knowledge 表添加 file_path 字段
ALTER TABLE knowledge 
ADD COLUMN IF NOT EXISTS file_path TEXT;

-- 添加注释
COMMENT ON COLUMN knowledge.file_path IS '.md 文件存储路径';

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_knowledge_file_path ON knowledge(file_path);
```

**执行命令**:
```bash
docker exec -i memory-hub-db psql -U memory_user -d memory_hub \
  < /home/wen/projects/memory-hub/database/04_add_file_path_to_knowledge.sql
```

**验证结果**: ✅ 字段已添加，索引已创建，查询性能优化

# 任务卡片系统

## 1. 系统概述

### 为什么需要任务卡片

大模型存在**失忆问题**：对话上下文有限，长任务容易忘记细节。任务卡片系统通过**持久化存储**解决这一问题。

### 任务卡片的作用

| 作用 | 说明 |
|------|------|
| **记忆持久化** | 任务信息写入文件，不怕上下文丢失 |
| **进度追踪** | 实时记录执行进度，随时可查看 |
| **责任明确** | 明确负责人、项目地址、截止时间 |
| **问题记录** | 遇到的问题和解决方案都有记录 |
| **可追溯** | 完整记录任务从开始到结束的全过程 |

### 任务卡片的位置

```
项目目录/TASK_CURRENT.md
```

**示例**：
```
/home/wen/projects/memory-hub/TASK_CURRENT.md
```

## 2. 任务卡片结构

```markdown
# 当前任务 - [手下名字]

**任务开始时间**: YYYY-MM-DD HH:MM
**优先级**: 高/中/低
**截止时间**: YYYY-MM-DD HH:MM

---

## 任务描述
[详细描述任务内容]

---

## 项目信息
**项目地址**: `/path/to/project`
**负责人**: [手下名字]

---

## 下一步行动
- [ ] 阅读本文件，确认任务信息
- [ ] 开始执行任务
- [ ] 完成后更新本文件

---

## 进度记录
YYYY-MM-DD HH:MM - [进度说明]

---

## 问题日志
[遇到的问题及解决方案]

---

## 注意事项
1. 代码写到项目目录，不是工作区
2. 每完成一个功能，更新"进度记录"
3. 遇到问题记录在"问题日志"
4. 任务完成后运行 task-on-complete.sh
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| 任务开始时间 | ✅ | 任务创建时间 |
| 优先级 | ✅ | 高/中/低，决定执行顺序 |
| 截止时间 | ✅ | 任务完成期限 |
| 任务描述 | ✅ | 详细任务内容 |
| 项目地址 | ✅ | 代码/文件存放位置 |
| 负责人 | ✅ | 执行任务的手下 |
| 进度记录 | ✅ | 按时间顺序记录进度 |
| 问题日志 | ⭕ | 有问题的话记录 |

## 3. Hook 机制

### task-on-start.sh

**功能**：
- 生成任务卡片
- 记录任务信息（描述、负责人、项目地址、优先级、截止时间）
- 持久化到项目目录

**调用时机**：傻妞派任务时自动调用

### task-on-complete.sh

**功能**：
- 更新任务状态为"已完成"
- 记录完成时间和结果
- 可选：同步到飞书多维表格

**调用时机**：手下完成任务后自动调用

## 4. 工作流程

```
1. 傻妞派任务 → 调用 task-on-start.sh
   ↓
2. 创建 TASK_CURRENT.md
   ↓
3. 派给手下（子 agent 模式）
   ↓
4. 手下阅读 TASK_CURRENT.md
   ↓
5. 手下执行任务
   ↓
6. 手下更新 TASK_CURRENT.md
   ↓
7. 任务完成 → 调用 task-on-complete.sh
```

## 5. 脚本使用示例

### 创建任务

```bash
bash /home/wen/.openclaw/workspace/hooks/task-on-start.sh \
  "修复 API 超时问题" \
  "小码" \
  "/home/wen/projects/memory-hub/" \
  "高" \
  "2026-03-07 18:00"
```

**参数说明**：
| 参数 | 说明 |
|------|------|
| 参数 1 | 任务描述 |
| 参数 2 | 负责人（手下名字） |
| 参数 3 | 项目地址 |
| 参数 4 | 优先级（高/中/低） |
| 参数 5 | 截止时间 |

### 完成任务

```bash
bash /home/wen/.openclaw/workspace/hooks/task-on-complete.sh \
  "/home/wen/projects/memory-hub/" \
  "已修复 LLM 配置，API 响应正常"
```

**参数说明**：
| 参数 | 说明 |
|------|------|
| 参数 1 | 项目地址 |
| 参数 2 | 完成说明 |

## 6. 飞书多维表格集成

### 表格字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 任务 ID | 文本 | 自动生成，唯一标识 |
| 任务描述 | 文本 | 任务详细内容 |
| 负责人 | 单选 | 小码/小搜/小写/小审/小析/小览/小图/小排 |
| 项目地址 | 文本 | 项目目录路径 |
| 优先级 | 单选 | 高/中/低 |
| 开始时间 | 日期时间 | 任务创建时间 |
| 截止时间 | 日期时间 | 任务完成期限 |
| 状态 | 单选 | 待办/进行中/已完成/已超时 |
| 结果说明 | 文本 | 任务完成后的说明 |

### 查看方式

1. 打开飞书多维表格
2. 筛选"进行中"任务
3. 查看任务详情
4. 可按负责人、优先级、截止时间排序

### 同步机制

```
task-on-complete.sh
       ↓
调用飞书 API
       ↓
创建/更新记录
       ↓
飞书多维表格
```

## 7. 超时处理

### 超时定义

**超过 5 分钟未完成**即视为超时

### 处理流程

```
1. Hook 脚本检测超时
   ↓
2. 记录到问题日志
   ↓
3. 傻妞收到通知
   ↓
4. 优化或重新派单
```

### 超时通知

```markdown
⚠️ 任务超时通知

**任务**：[任务描述]
**负责人**：[手下名字]
**超时时间**：[超时时长]
**项目地址**：[项目路径]

请傻妞处理：
- [ ] 联系手下了解情况
- [ ] 重新评估任务难度
- [ ] 调整或重新派单
```

## 9. 常见问题

### Q1：手下忘记任务怎么办？

→ **任务卡片持久化，随时可 read**

手下只需记住触发词："查看任务卡片信息（位置：项目目录/TASK_CURRENT.md）"

### Q2：任务卡片太多怎么办？

→ **定期归档已完成的任务**

建议每周归档一次，将已完成的 TASK_CURRENT.md 移动到 `docs/task-archive/` 目录。

```bash
# 归档示例
mkdir -p docs/task-archive
mv TASK_CURRENT.md docs/task-archive/TASK_$(date +%Y%m%d_%H%M%S).md
```

### Q3：飞书表格同步失败怎么办？

→ **检查飞书应用权限和配置**

排查步骤：
1. 检查飞书应用是否有写入权限
2. 检查 app_token 和 table_id 是否正确
3. 检查网络连接是否正常
4. 查看 Hook 脚本日志，定位错误信息

### Q4：任务卡片被意外删除怎么办？

→ **从飞书多维表格恢复**

飞书表格作为备份，可根据表格记录重新创建任务卡片。

### Q5：多个任务同时进行怎么办？

→ **每个项目独立 TASK_CURRENT.md**

不同项目在不同目录，各自的任务卡片互不干扰。傻妞负责协调优先级。

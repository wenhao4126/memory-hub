# 多智能体协作闭环

## 1. 系统概述

本系统采用**CEO+ 团队**的协作模式，通过傻妞（CEO+ 产品经理）统一调度 8 个专业手下，实现任务的高效分工与执行。

### 傻妞的角色定位

傻妞是整个系统的**大脑和中枢**，承担两个核心角色：

- **CEO（首席执行官）**：负责整体决策、任务分配、结果审核
- **产品经理**：负责理解需求、拆解任务、质量把控

傻妞的核心原则：**只动口不动手**，专注于任务拆解和结果审核，具体执行交给专业手下。

| 代号 | 职责 | 技能方向 |

| 小搜 🟢 | 信息采集 | 搜索、爬虫、数据收集 |

| 小码 🟡 | 代码开发 | 编程、调试、重构 |

| 小排 📐 | 内容排版 | 格式调整、排版优化 |

## 3. 协作流程

### 完整闭环

```
憨货需求 → 傻妞理解 → 任务拆解 → 派给手下 → 
手下执行 → 提交结果 → 傻妞审核 → 交付憨货
```

### 详细步骤

| 步骤 | 执行者 | 动作 | 产出 |
|------|--------|------|------|
| 1 | 憨货 | 提出需求 | 需求描述 |
| 2 | 傻妞 | 理解意图，拆解成子任务 | 任务清单 |
| 3 | 傻妞 | 根据技能匹配，派给合适的手下 | 任务分配 |
| 4 | 手下 | 阅读任务卡片（TASK_CURRENT.md） | 任务理解 |
| 5 | 手下 | 执行任务 | 任务结果 |
| 6 | 手下 | 更新任务卡片 | 进度记录 |
| 7 | 傻妞 | 审核结果 | 审核意见 |
| 8 | 傻妞 | 交付给憨货 | 最终交付物 |

## 5. 技能分配原则

### 核心原则

1. **按需分配**：不占茅坑不拉屎，有任务才激活技能
2. **专业专注**：每个手下 2-9 个技能，不贪多
3. **独立存放**：技能放在 `~/.openclaw/workspace-team-xxx/skills/`

## 6. 实际案例

### 案例 1：憨货要搜 AI 新闻

```
憨货："帮我搜一下最新的 AI 新闻"
        ↓
傻妞：理解需求 → 派给小搜 🟢
        ↓
小搜：使用 agent-reach 技能搜索
        ↓
提交结果 → 傻妞审核 → 交付憨货
```

**执行细节**：
- 傻妞：识别"搜索"需求，匹配小搜的技能
- 小搜：调用 agent-reach 技能，搜索 Twitter/知乎/新闻网站
- 傻妞：检查结果质量，整理后交付憨货

### 案例 2：憨货要修复 Bug

```
憨货："API 超时了，帮我看看"
        ↓
傻妞：理解需求 → 派给小码 🟡
        ↓
小码：检查日志、修复代码
        ↓
提交结果 → 傻妞审核 → 交付憨货
```

**执行细节**：
- 傻妞：识别"修复 Bug"需求，匹配小码的技能
- 小码：查看日志、定位问题、修复代码、测试验证
- 小审：可选介入，进行代码审查
- 傻妞：确认修复效果，交付憨货

### 案例 3：憨货要写小红书文案

```
憨货："帮我写个 AI 工具的小红书种草文案"
        ↓
傻妞：理解需求 → 派给小写 🟢
        ↓
小写：brainstorming → 撰写文案 → 自改 3 遍
        ↓
提交结果 → 傻妞审核 → 交付憨货
```

**执行细节**：
- 傻妞：识别"文案撰写"需求，匹配小写的技能
- 小写：先用 brainstorming 明确需求，再用 marketing-psychology 优化文案
- 傻妞：检查文案风格、错别字、平台适配性
- 小排：可选介入，优化排版格式

## 7. 优势总结

| 优势 | 说明 |
|------|------|
| **专业分工** | 每个手下有自己的专长，术业有专攻 |
| **任务持久化** | 任务卡片记录一切，不怕大模型失忆 |
| **质量把关** | 傻妞审核后才交付，保证质量 |
| **可追溯** | 任务卡片记录完整过程，随时可查 |
| **灵活扩展** | 按需增加手下或技能，不影响整体架构 |
| **高效协作** | 并行处理多个任务，效率最大化 |

## 8. 常见问题

### Q1：手下太多会不会混乱？
→ 不会。傻妞统一调度，任务卡片明确责任，每个任务只有一个负责人。

### Q2：手下之间如何协作？
→ 复杂任务可拆分成多个子任务，分别派给不同手下，傻妞负责整合结果。

### Q3：手下执行失败怎么办？
→ 记录到问题日志，傻妞评估后决定：重新派单/换人/调整任务。

### Q4：如何保证任务不超时？
→ Hook 脚本检测超时，超时自动通知傻妞，傻妞决定优化或重新派单。

### ✅ 问题 2: API 端点错误 - /api/v1/agents

**问题描述**: `/api/v1/agents` 返回 "Internal Server Error"

**根因**: 
- Pydantic 验证错误：`capabilities` 字段为 `None` 时无法通过验证
- 数据库中旧数据的 `capabilities` 字段为 `NULL`

**修复方案**:
- 修改 `backend/app/api/routes.py` 的 `list_agents()` 函数
- 在处理数据库返回数据时，将 `None` 值转换为空列表/空字典

**修改文件**:
- `backend/app/api/routes.py` - 添加 `None` 值处理逻辑

**代码变更**:
```python
# 修复前
if isinstance(data.get('capabilities'), str):
    data['capabilities'] = json.loads(data['capabilities'])

# 修复后
if isinstance(data.get('capabilities'), str):
    data['capabilities'] = json.loads(data['capabilities'])
elif data.get('capabilities') is None:
    data['capabilities'] = []

# 同样处理 metadata 字段
```

**验证结果**:
```bash
curl http://localhost:8000/api/v1/agents
# ✅ 返回智能体列表，不再报错
```

**状态**: ✅ 已修复并验证

### ⚠️ 问题 2b: API 端点错误 - /api/v1/memories/search

**问题描述**: `/api/v1/memories/search/text` 返回 "服务器内部错误"

**根因**: 
- 向量维度不匹配：期望 1536 维，实际生成 1024 维
- 配置文件中 `VECTOR_DIMENSIONS=1536`，但 Embedding 模型使用 `text-embedding-v4`（1024 维）

**修复方案**:
- 修改 `.env` 文件中的 `VECTOR_DIMENSIONS=1024`
- 或者修改 Embedding 模型为 `text-embedding-v2`（1536 维）

**建议**:
- 推荐使用 `text-embedding-v4`（1024 维），更新配置即可
- 这是配置问题，不是代码 bug

**状态**: ⚠️ 配置问题，需要更新 .env 文件
